One command installs WSL + Ubuntu. This takes 2-5 minutes and requires a restart.
POWERSHELL (ADMIN)

‚ö†Ô∏è Restart required. After the command completes, restart your computer. WSL finishes setup on reboot.

After restart, Ubuntu opens automatically. Create a username and password (this is your Linux login, not your Windows one).
‚ÑπÔ∏è If Ubuntu doesn't open automatically: search "Ubuntu" in Start menu, or type wsl in PowerShell.

Troubleshoot: "WSL 2 requires an update to its kernel component"

Download and install the WSL2 kernel update package:

Download WSL2 kernel update (Microsoft)

Run the MSI installer, then restart and try wsl again.

Troubleshoot: "Virtualization must be enabled in BIOS"

WSL2 requires hardware virtualization. Restart your PC, enter BIOS (usually F2, F12, or DEL during boot), and enable:

Intel: "Intel VT-x" or "Intel Virtualization Technology"

AMD: "AMD-V" or "SVM Mode"

Microsoft: Enable virtualization

‚úÖ Expected: You see "Ubuntu" and "Linux" in the output. You're now running Linux inside Windows.

Troubleshoot: "Permission denied" during npm install

Fix npm global permissions:

BASH
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @anthropic-ai/claude-code
Troubleshoot: Browser doesn't open for auth

WSL may not auto-open browsers. Copy the URL from terminal output and paste it into your Windows browser manually. After authorizing, the CLI will detect the auth token.

üöÄ Phase 4 ‚Äî Launch Claude Code Inside tmux
9
Enable Agent Teams experimental flag
Set the environment variable so Claude Code unlocks Agent Teams capability.
WSL / BASH 
# Add to .bashrc so it persists across sessions
echo 'export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1' >> ~/.bashrc
source ~/.bashrc

# Verify it's set
echo $CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS
‚úÖ Expected output: 1
‚ÑπÔ∏è Our repos already have .claude/settings.local.json with this flag ‚Äî but the env var ensures it works everywhere.
10
Navigate to your project repo
cd into the repo where you want Agent Teams to work. For county scrapers, use zonewise-modal.
WSL / BASH 
# Clone if you haven't already
git clone https://github.com/breverdbidder/zonewise-modal.git ~/zonewise-modal

# Navigate into it
cd ~/zonewise-modal
11
Start tmux session, then launch Claude Code
This is the critical step. Claude Code MUST be running inside tmux for Agent Teams split panes to work.
WSL / BASH ‚Äî RUN THESE IN ORDER 
# Step A: Start a tmux session named "zonewise"
tmux new-session -s zonewise

# Step B: (you're now INSIDE tmux ‚Äî notice the green bar at bottom)
# Launch Claude Code from within tmux
claude
‚ö†Ô∏è Order matters! tmux first, THEN claude inside it. If you run claude outside tmux, agents spawn headless with no visible panes.
‚úÖ How to know you're in tmux: Look for a green status bar at the bottom of your terminal showing [zonewise].
Troubleshoot: "sessions should be nested with care"

You're already inside a tmux session and trying to create another. Exit first:

BASH
# Detach from current session
# Press: Ctrl+B, then D

# Kill any existing sessions
tmux kill-server

# Start fresh
tmux new-session -s zonewise
Troubleshoot: tmux colors look broken or garbled

Add 256-color support:

BASH
echo 'set -g default-terminal "screen-256color"' >> ~/.tmux.conf
tmux source-file ~/.tmux.conf

‚ö° tmux Cheat Sheet
tmux new-session -s zonewiseCreate new named session
Ctrl+B ‚Üí DDetach from tmux (agents keep running in background)
tmux attach -t zonewiseReattach to running session
tmux lsList all active sessions
Ctrl+B ‚Üí Arrow keysSwitch between agent panes
Ctrl+B ‚Üí ZZoom/fullscreen a single pane (press again to unzoom)
Ctrl+B ‚Üí [Scroll mode (use arrows/PgUp/PgDn, press Q to exit)
